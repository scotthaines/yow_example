Page example

A 520 x 390 image fits on a 1024 x 768 display with top of page text.

Introduction

This example page describes and shows the layout of topic pages. It helps you create new pages. Reference it for page layout and "Copy cannibalize" it to start new pages.

Centered and fixed width

Pages are centered and fixed width to give consistent displays at various resolutions. The styles.css sets this.

Page width

The page width is 960 pixels to fit on 1024 x 768 displays with spare room for the web browser's scroll bar. The styles.css sets this.

Single column

A single column of text keeps the layout simple. The styles.css sets this.

"Copy cannibalize" Page example

Copy the whole ...\topics\example_page folder and rename it for your new page. In your new page's topics folder edit topics.html. Change its title, header line, update date and content for your new page.

Links

Links mimicing Wikipedia. On-site links are inline. Off-site links are in footnotes. Picture size limits is an example of an inline link. The Max Design CSS layouts[1] is an example of an external reference. External references are done with footnotes.

Pictures

To see the whole image there are various picture size limits depending on page placement and the desired width to height ratio, for example, 1: 1, 4 : 3 or "wide screen" 16 : 9.

Picture descriptions are below the picture.

The maximum picture's dimensions seen completely in a browser on a 1024 x 768 display are about 880 wide by 600 high. If 1024 x 768 is the lowest resolution the page layout is for then the following are practical picture dimension limits:
